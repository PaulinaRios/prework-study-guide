HTML

  • HTML handles the content, as well the structure and the meaning of every element on a website.
  • HTML elements work together to provide content and the basic framework.
  • The head element contains information about the web page.
  • The body element represents the visible content shown to the user.
  • Elements are usually composed of an opening and closing tag represented with angle brakes.
  • The opening and closing tag contains everything that we want to include in the element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is used when a developer wants to style an element directly in the HTML file.
  • Inline styles are useful for quick and permanent changes but are less flexible than external and internal style sheets.
  • Internal style sheets contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS style sheet allows developers to keep all their CSS rules in a separate file.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Git commands are not the same as the bash commands, these commands always satrt with git and are used to save and track versions of your code.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds modifications in the current working branch.
  • git pull origin main: receives a branch's modifications into the local environment.
